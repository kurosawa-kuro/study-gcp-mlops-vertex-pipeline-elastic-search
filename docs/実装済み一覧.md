## ディレクトリ構成

```
study-gcp-mlops/
├── .github/workflows/      # CI/CD（GitHub Actions）
├── docs/                   # ドキュメント
├── makefiles/              # Makefile分割モジュール
├── scripts/                # 自動化スクリプト
├── src/
│   ├── api/                # Cloud Run Service（FastAPI推論API）
```

## ファイル一覧

### ルート

| ファイル | 説明 |
|---------|------|
| `.env.example` | 環境変数テンプレート（GCS・ML・Discord） |
| `.gitignore` | 除外設定（credentials, tfstate, mlruns等） |
| `CLAUDE.md` | Claude Code向けプロジェクト説明書 |
| `Makefile` | ビルド・デプロイコマンド統合エントリポイント |
| `README.md` | プロジェクト概要・技術スタック・構成説明 |

### .github/workflows/

| ファイル | 説明 |
|---------|------|
| `batch-deploy.yml` | batch CI/CD（test→build→push→Cloud Run Job更新） |
| `api-deploy.yml` | API CI/CD（test→build→push→Cloud Run Service更新） |
| `terraform.yml` | Terraform CI/CD（PR→plan表示、main→apply自動実行） |

### docs/

| ファイル | 説明 |
|---------|------|
| `仕様、設計書.md` | アーキテクチャ・Terraform設計・デプロイフロー詳細 |
| `実装済み一覧.md` | 本ファイル |

### makefiles/

| ファイル | 説明 |
|---------|------|
| `api.mk` | FastAPI APIのテスト・ビルド・デプロイコマンド |
| `batch.mk` | ML学習パイプラインのテスト・ビルド・デプロイ・MLflow UI |
| `gcp.mk` | GCP API有効化・SA権限・Docker認証設定 |
| `terraform.mk` | Terraform init/plan/apply/destroy/fmt |

### scripts/

| ファイル | 説明 |
|---------|------|
| `core.py` | 共通ユーティリティ（定数・JSON構造化ログ・load_env・run・notify_discord） |
| `deploy_all.py` | 全体冪等デプロイ（batch+API→実行） |
| `monitor_batch.py` | Cloud Run Job実行結果監視＋Discord失敗通知 |
| `monitor_api.py` | API健全性チェック（/health）＋Discord異常通知 |
| `check_drift.py` | モデルドリフト検知（RMSE閾値チェック＋Discord通知） |
| `reset_all.py` | GCP全リソース削除＆ローカル・ML成果物クリーン |
| `setup_gcp.sh` | Google Cloud CLIインストール・初期設定 |
| `setup_terraform.sh` | Terraformインストール・初期設定 |

### src/api/（FastAPI推論API）

| ファイル | 説明 |
|---------|------|
| `Dockerfile` | Python 3.11-slim + FastAPI コンテナ定義 |
| `main.py` | 推論API（BigQuery最良モデル取得→GCSロード→POST /predict） |
| `requirements.txt` | 依存（FastAPI・uvicorn・scikit-learn・bigquery・gcs） |
| `requirements-dev.txt` | 開発依存（pytest・httpx） |
| `test_main.py` | GCSパース・ヘルスチェック・推論・エラー系テスト（6件） |

### src/batch/（ML学習パイプライン）

| ファイル | 説明 |
|---------|------|
| `Dockerfile` | Python 3.11-slim＋データ事前キャッシュ コンテナ定義 |
| `dataset.py` | California Housing取得＆train/test分割 |
| `train.py` | RandomForest構築・学習・評価（RMSE・MAE） |
| `model_store.py` | モデルのローカル/GCS保存・読み込み |
| `bq_store.py` | BigQueryへの評価メトリクス投入 |
| `main.py` | パイプライン（データ→学習→評価→MLflow→GCS保存→ログ出力→BQ投入） |
| `requirements.txt` | 依存（scikit-learn・pandas・gcs・bigquery・mlflow） |
| `requirements-dev.txt` | 開発依存（pytest） |
| `test_main.py` | データ・学習・保存・ログ・MLflow・BQ統合テスト（13件） |
| `test_e2e.py` | エンドツーエンドテスト（学習→保存→BQ→推論一周、3件） |

### terraform/

| ファイル | 説明 |
|---------|------|
| `artifact_registry.tf` | Docker形式Artifact Registryリポジトリ定義 |
| `bigquery.tf` | BigQueryデータセット（mlops）・metricsテーブル（90日リテンション・日別パーティション） |
| `cloud_run_job.tf` | Cloud Run Job（ml-batch）定義＋環境変数（BQ_DATASET含む） |
| `cloud_run_service.tf` | Cloud Run Service（ml-api）定義＋公開IAM |
| `cloud_scheduler.tf` | Cloud Scheduler cron定期実行（毎日9:00 JST） |
| `gcs.tf` | GCSデータバケット定義（uniform access） |
| `iam.tf` | GCS書き込み/読み取り・BigQuery編集・Scheduler実行権限付与 |
| `provider.tf` | Google provider v5・credentials認証設定 |
| `variables.tf` | プロジェクト・リージョン・リポジトリ・バケット名 |
