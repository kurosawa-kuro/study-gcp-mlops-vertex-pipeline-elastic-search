# Elastic Search 仕様・設計書

最終更新: 2026-04-04

---

## 1. 概要

Cloud Run Job から Elastic Cloud（Elasticsearch）へ接続し、ドキュメント投入・検索・クリーンアップを行うバッチジョブ。
Terraform で全リソースを管理する。

---

## 2. アーキテクチャ

```
[WSL]                    [GCP]
  │
  ├─ docker build
  │
  ├─ docker push ──────► Artifact Registry (hello-elastic)
  │                           │
  ├─ gcloud run jobs ────────►│
  │   create                  │
  │                           ▼
  ├─ gcloud run jobs     Cloud Run Job (es-hello)
  │   execute ──────────► (コンテナ起動)
  │                           │
  │                      Secret Manager (elastic-api-key)
  │                           │
  │                           ▼
  │                      Elastic Cloud (asia-northeast1)
  │                      (ES接続・検索)
  │                           │
  └─ gcloud logging ◄─────────┘
      read (ログ確認)
```

---

## 3. 処理フロー

```
Cloud Run Job (es-hello)
  ├── 1. 疎通確認（es.info）
  ├── 2. インデックス作成 + ドキュメント投入
  ├── 3. 検索（match query）
  └── 4. クリーンアップ（インデックス削除）
```

ソースコード: `src/elastic-search/`（main.py, requirements.txt, Dockerfile）

## 5. Terraform リソース定義

### リソース対応表

```
手動でやったこと                    → Terraform resource
──────────────────────────────────────────────────────
Elastic Cloud デプロイメント作成     → ec_deployment
Artifact Registry リポジトリ作成    → google_artifact_registry_repository
Secret Manager シークレット作成     → google_secret_manager_secret
                                      google_secret_manager_secret_version
Cloud Run Job 作成                  → google_cloud_run_v2_job
IAM 権限付与                        → google_secret_manager_secret_iam_member
```

### ディレクトリ構成

```
terraform/
├── main.tf          # リソース定義
├── variables.tf     # 変数定義
├── outputs.tf       # 出力値
├── providers.tf     # provider設定 (elastic/ec + hashicorp/google)
└── terraform.tfvars # 変数の実値 (gitignore必須)
```

### Provider

```hcl
terraform {
  required_providers {
    ec = {
      source  = "elastic/ec"
      version = "~> 0.10"
    }
    google = {
      source  = "hashicorp/google"
      version = "~> 5.0"
    }
  }
}

provider "ec" {
  apikey = var.elastic_cloud_api_key
}

provider "google" {
  project = var.project_id
  region  = var.region
}
```

### 主要リソース

```hcl
# Elastic Cloud デプロイメント
resource "ec_deployment" "hello" {
  name                   = "my-deployment"
  region                 = "gcp-asia-northeast1"
  version                = "9.3.2"
  deployment_template_id = "gcp-storage-optimized"

  elasticsearch = {
    hot = {
      autoscaling = {}
      size        = "1g"
      zone_count  = 1
    }
  }

  kibana = {
    zone_count = 1
  }
}

# Cloud Run Job
resource "google_cloud_run_v2_job" "hello" {
  name     = "es-hello"
  location = var.region

  template {
    template {
      containers {
        image = "${var.region}-docker.pkg.dev/${var.project_id}/hello-elastic/es-hello:latest"

        env {
          name  = "ELASTIC_CLOUD_URL"
          value = ec_deployment.hello.elasticsearch[0].https_endpoint
        }
        env {
          name = "ELASTIC_API_KEY"
          value_source {
            secret_key_ref {
              secret  = google_secret_manager_secret.elastic_api_key.secret_id
              version = "latest"
            }
          }
        }
      }
    }
  }

  depends_on = [google_secret_manager_secret_iam_member.hello]
}
```

### Outputs

```hcl
output "elasticsearch_endpoint" {
  value = ec_deployment.hello.elasticsearch[0].https_endpoint
}

output "cloud_run_job_name" {
  value = google_cloud_run_v2_job.hello.name
}

output "artifact_registry_url" {
  value = "${var.region}-docker.pkg.dev/${var.project_id}/hello-elastic"
}
```

## 6. デプロイ手順（手動）

Terraform未使用時の手動デプロイ手順。

```bash
# 環境変数
export PROJECT_ID=mlops-dev-a
export REGION=asia-northeast1
export IMAGE_NAME=es-hello
export REPO_NAME=hello-elastic

# 1. Artifact Registry リポジトリ作成
gcloud artifacts repositories create ${REPO_NAME} \
  --repository-format=docker --location=${REGION}

# 2. Docker認証 → ビルド → push
gcloud auth configure-docker ${REGION}-docker.pkg.dev
docker build -t ${REGION}-docker.pkg.dev/${PROJECT_ID}/${REPO_NAME}/${IMAGE_NAME}:latest .
docker push ${REGION}-docker.pkg.dev/${PROJECT_ID}/${REPO_NAME}/${IMAGE_NAME}:latest

# 3. Secret Manager にAPIキー登録
echo -n "${ELASTIC_API_KEY}" | \
  gcloud secrets create elastic-api-key \
  --data-file=- --replication-policy=user-managed --locations=${REGION}

# 4. Cloud Run Job 作成 → 実行
gcloud run jobs create ${IMAGE_NAME} \
  --image=${REGION}-docker.pkg.dev/${PROJECT_ID}/${REPO_NAME}/${IMAGE_NAME}:latest \
  --region=${REGION} \
  --set-env-vars="ELASTIC_CLOUD_URL=${ELASTIC_CLOUD_URL}" \
  --set-secrets="ELASTIC_API_KEY=elastic-api-key:latest" \
  --task-timeout=60s --max-retries=0

gcloud run jobs execute ${IMAGE_NAME} --region=${REGION} --wait

# 5. ログ確認
gcloud logging read \
  "resource.type=cloud_run_job AND resource.labels.job_name=${IMAGE_NAME}" \
  --limit=50 --format="value(textPayload)"
```

## 7. トラブルシューティング

### Artifact Registry push の permission denied

```bash
gcloud projects add-iam-policy-binding ${PROJECT_ID} \
  --member="user:$(gcloud config get-value account)" \
  --role="roles/artifactregistry.writer"
```

### Secret Manager アクセスエラー

```bash
PROJECT_NUMBER=$(gcloud projects describe ${PROJECT_ID} --format="value(projectNumber)")
gcloud secrets add-iam-policy-binding elastic-api-key \
  --member="serviceAccount:${PROJECT_NUMBER}-compute@developer.gserviceaccount.com" \
  --role="roles/secretmanager.secretAccessor"
```

### API 未有効化

```bash
gcloud services enable \
  run.googleapis.com \
  artifactregistry.googleapis.com \
  secretmanager.googleapis.com
```

## 8. クリーンアップ

```bash
# Terraform管理の場合
terraform destroy -var-file="terraform.tfvars"

# 手動管理の場合
gcloud run jobs delete ${IMAGE_NAME} --region=${REGION} --quiet
gcloud artifacts docker images delete \
  ${REGION}-docker.pkg.dev/${PROJECT_ID}/${REPO_NAME}/${IMAGE_NAME}:latest --quiet
gcloud secrets delete elastic-api-key --quiet
# Elastic Cloud: Console > My deployment > Actions > Delete deployment
```
