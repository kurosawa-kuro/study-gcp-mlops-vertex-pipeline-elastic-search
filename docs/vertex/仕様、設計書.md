# Vertex AI 仕様・設計書

最終更新: 2026-04-04

---

## 1. 概要

Vertex AI の主要機能を学習・検証するためのドキュメント。
本プロジェクトでは Vertex Pipelines を中心に採用し、Vertex Training / Vertex Endpoints は不採用とする。

---

## 2. 機能採用判断

| Vertex機能 | 判断 | 理由 |
|-----------|------|------|
| Vertex Pipelines (KFP) | 採用 | オーケストレーション可視化。→ 詳細は `vertex-pipeline/仕様、設計書.md` |
| Vertex Model Registry | 軽く採用 | GCS保存と併用。カタログとして活用 |
| Vertex Experiments | 不採用 | MLflow維持。ベンダー中立性を重視 |
| Vertex Endpoints | MVP実装済み | Notebook で deploy→推論→クリーンアップ確認済み。本番APIはCloud Run Serviceを維持 |
| Vertex Metadata | 自動取得のみ | KFPアーティファクト型指定で自動記録される |

---

## 3. 現行プロジェクトとの対応

| 現行（Cloud Run ベース） | Vertex AI | 備考 |
|-------------------------|-----------|------|
| `src/batch/train.py` で学習 | Custom Training Job | 同じsklearnコード。`AIP_MODEL_DIR`でGCS出力先を自動設定 |
| GCS に pickle 保存 | GCS保存 + Model Registry | バージョン管理が組み込み |
| BigQuery で最良モデル管理 | Model Registry | カタログ機能 |
| `src/api/main.py` FastAPI | Vertex AI Endpoint（MVP実装済み） | 本番APIはCloud Run Service維持 |
| MLflow メトリクス記録 | MLflow維持（Experiments不採用） | ベンダー中立性重視 |

---

## 4. 主要機能の概要

| 機能 | 概要 |
|------|------|
| Custom Training Job | 自前コンテナをVertex AI上で実行。`AIP_MODEL_DIR`でGCS出力先を自動設定 |
| Model Registry | GCSモデルを登録・バージョン管理。事前構築済みサービングコンテナで推論API自動提供 |
| Endpoint | モデルデプロイ先。オートスケーリング・トラフィック分割対応（本PJでは不採用） |
| Batch Prediction | エンドポイント不要で大量データ一括推論。JSONL入出力 |
| Pipelines (KFP v2) | ワークフロー自動化。DAG可視化・メタデータ追跡（本PJ採用） |

## 5. MVP: Notebookで体験する流れ（実装済み）

**実装ファイル:** `notebooks/vertex_housing_mvp.ipynb`（実行済み・クリーンアップ済み）

Notebook 1つでVertex AIの基本フロー（学習→Model Registry登録→エンドポイントデプロイ→推論）を体験する。パイプライン・コンテナビルド不要。

```
① ローカルで学習 → GCS にモデル保存 (gs://mlops-dev-a-vertex/models/california-housing-mvp/)
② Model Registry に登録（事前構築済みsklearnサービングコンテナ使用）
③ エンドポイントにデプロイ → 推論
④ クリーンアップ（Endpoint削除・Model削除・GCS削除）
```

### 実行結果

| 項目 | 値 |
|------|-----|
| RMSE | 0.5445 |
| MAE | 0.3663 |
| 予測例 | $437,111（入力: MedInc=8.3252, HouseAge=41.0, ...） |
| GCSバケット | `mlops-dev-a-vertex` |
| クリーンアップ | 完了（Endpoint・Model・GCS全削除済み） |

## 7. 実装リファレンス

### Model Registry 登録

```python
from google.cloud import aiplatform

aiplatform.init(project="mlops-dev-a", location="asia-northeast1")

model = aiplatform.Model.upload(
    display_name="california-housing-rf",
    artifact_uri="gs://mlops-dev-a-vertex/models/california-housing/",
    serving_container_image_uri="us-docker.pkg.dev/vertex-ai/prediction/sklearn-cpu.1-3:latest",
)
```

### Endpoint デプロイ & 推論

```python
endpoint = model.deploy(
    deployed_model_display_name="california-housing-rf-v1",
    machine_type="n1-standard-2",
    min_replica_count=1,
    max_replica_count=3,
)

# 推論（8特徴量: MedInc, HouseAge, AveRooms, AveBedrms, Population, AveOccup, Latitude, Longitude）
result = endpoint.predict(instances=[
    [8.3252, 41.0, 6.984, 1.024, 322.0, 2.556, 37.88, -122.23],
])
# 例: predictions = [4.526]（単位: $100,000 → $452,600）
```

### Custom Training Job

```python
IMAGE_URI = "asia-northeast1-docker.pkg.dev/mlops-dev-a/vertex-training/california-housing:v1"

job = aiplatform.CustomContainerTrainingJob(
    display_name="california-housing-rf",
    container_uri=IMAGE_URI,
    model_serving_container_image_uri="us-docker.pkg.dev/vertex-ai/prediction/sklearn-cpu.1-3:latest",
)

model = job.run(
    args=["--n-estimators", "100", "--max-depth", "10"],
    replica_count=1,
    machine_type="n1-standard-4",
    model_display_name="california-housing-rf",
    base_output_dir="gs://mlops-dev-a-vertex/models/california-housing",
)
```

### Batch Prediction

```python
model = aiplatform.Model(model_name="california-housing-rf")

batch_prediction_job = model.batch_predict(
    job_display_name="california-housing-batch",
    instances_format="jsonl",
    predictions_format="jsonl",
    gcs_source="gs://mlops-dev-a-vertex/batch-input/input.jsonl",
    gcs_destination_prefix="gs://mlops-dev-a-vertex/batch-output/",
    machine_type="n1-standard-4",
)
```

### クリーンアップ（重要: Endpoint は常時課金）

```python
endpoint.undeploy_all()
endpoint.delete()
model.delete()
```
